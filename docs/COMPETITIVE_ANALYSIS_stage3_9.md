# 경쟁 분석 — 뮤블(Muvel) · 펜시브(Pensiv) vs Bindery (Stage 3.9)

기준일 2026-07-02. 뮤블 공식 가이드/커뮤니티 공개 자료, 펜시브 공개 소개 기준.

## 1. 기능 비교표

| 영역 | 뮤블 | 펜시브 | Bindery (현재) |
|---|---|---|---|
| 원고 구조 | 에피소드/플롯/위키/메모 4축 | 독스/캐릭터/플롯보드 | 에피소드 파이프라인(원고·컨텍스트·QA·수정) + Codex + Plot Grid |
| 저장 | 클라우드 기본 + 로컬(설치판), 실시간 자동저장, 버전 | 클라우드 | **로컬 Markdown 단일 진실원** + 자동저장 + 스냅샷(sha256) — 차별점 |
| 에디터 편의 | 스마트 따옴표, 자동 대치, 작가 주석(//), 타자기 스크롤, 위젯 | 에디터 중심 | ✅ 스마트 따옴표·자동 대치·`//`주석(3.9 반영), 타자기·포커스·Zen, 세션 목표 |
| AI | AI 리뷰/분석/요약/대체표현 (뮤블) | **검색+계획 통합 엔진**: 설정 충돌 검증, 첫 등장 회차 추적, 의도 확인 후 플랜 제안 | QA 게이트, 반복/상투 분석, 후보 A/B→diff→hunk 적용, 수정계획, 프롬프트 프리뷰 |
| 설정집 지능 | 위키(수동) | 문서 전체 검색·근거 주석 | Dynamic Link 스캔(신뢰도), 멘션 hover 카드, Progressions 타임라인, `[[링크]]` 삽입 |
| 플롯 | 플롯 에디터 | 플롯보드 | Plot Grid + 긴장도/서브플롯 공백 경고 (Board/Canvas 미구현) |
| 안전장치 | 버전 자동 저장 | — | **후보=diff 검토 후 적용, 적용 전 버퍼 스냅샷** — 가장 강함 |

## 2. Bindery가 앞서는 지점
1. **Evidence-first AI 하네스**: AI 출력이 절대 원고를 직접 덮지 않는 candidate→diff→hunk apply 체계. 뮤블·펜시브 모두 없는 안전 모델.
2. **로컬 Markdown**: git 친화, 종속성 없음(뮤블 커뮤니티의 '섭종 불안'이 원천 차단됨).
3. **QA 게이트 정량화**: Plot/Continuity/Style/Voice/Lexicon 점수 + 라인 점프 + 수정계획 연동.
4. **프롬프트 투명성**: 전송될 프롬프트를 실행 전 열람(3.8) — 펜시브의 '의도 먼저 확인' 철학을 UI로 구현.

## 3. 부족한 지점 → 개선계획
| 항목 | 대상 벤치마크 | 상태 |
|---|---|---|
| 스마트 따옴표·자동 대치·작가 주석 | 뮤블 | ✅ **3.9 반영** (settings 토글, IME 안전) |
| "이 인물 첫 등장이 몇 화?" 근거 추적 | 펜시브 | 부분(Progressions·firstAppearance) → 회차 코퍼스 인덱싱 필요 (M) |
| 설정 충돌 자동 대조 | 펜시브 | QA continuity 게이트가 자리, 규칙 엔진은 로드맵 (L) |
| 위젯 시스템(글자수/타이머/사전 등 배치형) | 뮤블 | 대체: 우측 탭 구조. 위젯화는 선택 로드맵 (L) |
| 회차 요약 누적(장기 컨텍스트 하네스) | 펜시브 | summarize 산출물을 context 팩에 자동 편입하는 체인 (M) |
| 에피소드 드래그 정렬 / Plot Board | 뮤블·펜시브 | 미구현 (L) |
| 본문 폰트 선택(바탕/명조/고딕)·크기 | 뮤블 프리셋 | 미구현 (S) — 다음 단계 최우선 |

## 4. UI/UX·폰트 평가 (3.8 재설계 이후)
- 라이트 웜페이퍼 + 화면당 primary 1개 + 기능 탭 분리: 뮤블의 "시야 정리" 원칙과 정합.
- 본문 세리프(Noto Serif KR 스택) 16px/2.0/70ch — 플랫폼 열람감. 개선 여지: 사용자 폰트 프리셋(문피아/시리즈풍), 문단 들여쓰기 옵션.
- 다크는 검토·야간용 보조 테마로 유지.

## 5. AI 장기 집필 하네스 — 현재 아키텍처 요약
`회차 열기 → (Context 팩) → Draft A/B 후보 → 실텍스트 Analyze → QA 게이트 → Revision 체크리스트 → 후보 diff/hunk 적용(사전 스냅샷) → Commit`.
장기 연재 관점 보강 순서: ① summarize 누적→context 자동 주입, ② 코퍼스 전회차 인덱스(첫 등장/설정 근거), ③ 충돌 규칙 엔진.
