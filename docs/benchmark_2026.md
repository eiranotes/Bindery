# Benchmark 강화 메모 — 2026-07-01

## Muvel

뮤블은 에피소드/플롯/위키/메모 구조, 플롯 캔버스, 다이나믹 링크, 반복 표현 지도, 에피소드 분석/리뷰를 기준 기능으로 삼는다. Bindery는 이를 Markdown-native로 재해석한다.

## Scrivener

Scrivener의 Binder/Corkboard/Outliner 연동은 Binder tree, Episode tabs, Plot board/grid의 원형이다. 특히 구조 변경이 원고 구조와 연결되어야 한다.

## Novelcrafter

Codex는 source of truth 역할을 한다. Bindery의 `canon/`, `characters/`, `world/`, `progressions/`는 Codex 계층이며 Gemini 에이전트는 이 계층을 읽되 직접 파괴하지 않는다.

## Dabble / Plottr / Campfire

- Dabble: Plot Grid와 scene/manuscript sync.
- Plottr: scene cards, plotlines, scene attributes.
- Campfire: timeline, arcs, worldbuilding modules.

Bindery는 장면 frontmatter를 공통 원천으로 삼아 board/grid/canvas를 파생한다.

## Sudowrite

Sudowrite의 Write/Describe/Expand/Rewrite/Story Bible은 작성 보조 기능이다. Bindery에서는 `/continue`, `/expand`, `/rewrite` inline command 후보로 남긴다.

## Positioning

Bindery is not a cloud-first writing SaaS. It is a local, evidence-first, Markdown/Git-compatible fiction IDE for long serialized projects.
