# 간단 모드 사용자 흐름

작성: 2026-07-05 · 와이어프레임: Open Design `bindery-harness-ui/autopilot-shell.html`

## 메뉴

```text
홈 | 집필 | 작품노트 | 보류함 | 파일 | 설정
```

설계자 모드(설정에서 전환)에서는 소재·세계관·플롯·회차·제안·정사 화면과 run 도크가 추가된다.
전역 크롬은 상단바 + (선택) 하단 상태바뿐이다. 좌측 사이드바는 없다 — 최대 2단 규칙.
2단 분할은 작품노트(목록+뷰어) 한 곳만 쓴다.

## 흐름

### 1. 작품 시작 (홈)

```text
홈: "새 작품을 시작할 차례입니다."
  [한 줄 방향 입력(선택)] [AI에게 기획 맡기기]
→ 기획 후보 1개 카드 (제목·훅·독자 약속·위험)
→ [이 기획으로 시작]  ← 유일한 승인 지점 (작품 방향 확정)
→ 세계관 자산 반영 + 바이블 조립 + 플롯 초안 8화 자동 구성
```

### 2. 작품 기준 준비 (홈 또는 집필)

```text
바이블이 비어 있거나 현재 회차 플롯 row가 없음
→ 홈/집필: "작품 기준을 먼저 세울 차례입니다."
  [바이블과 플롯 준비하고 쓰기]
→ 작품노트·채택 자산에서 canon/setting-bible.md 조립
→ 그 바이블을 입력으로 plot/plot-board.json 정리
→ 집필 화면에서 설계부터 다시 진행
```

라이트모드에서도 집필은 `바이블 → 플롯 → 회차 설계 → 초안` 순서를 건너뛰지 않는다.
준비 전 초안 후보가 있더라도, 준비 후에는 설계부터 다시 잡아 새 후보를 만든다.

### 3. 회차 쓰기 (홈 → 집필)

```text
홈: "ep012를 이어 쓸 차례입니다." [한 줄 메모(선택)] [이번 화 쓰기]
→ 집필 화면에서 실시간 스트리밍 (설계 → 장면 → 집필안)
→ 집필안 카드 1개
   [미리보기(전문·diff)] [이 후보 적용]  ← hard commit, 스냅샷 선행
```

### 4. 검토·수정

```text
[원고 검토하고 수정안 만들기]
→ 문체·흐름·설정 점검 3종 병렬 실행 (각 실행 스트리밍)
→ 체크박스 수정 제안 (AI 추천 기본 체크)
→ [체크한 수정 적용]  ← hard commit, 스냅샷 선행
```

### 5. 마감

```text
[회차 마감]
→ 요약 + 다음 화 메모 + 설정 변경 후보 자동 생성
→ 마감 카드: 변경 후보 체크박스 (위험 높음만 기본 해제)
→ [체크한 항목 반영하고 마감] 또는 [보류]
   체크 해제 항목은 보류함으로. 픽스 + 재개 상태 갱신.
```

### 6. 보류함

```text
설정 변경·세계관 추가 제안을 체크박스로 일괄 검토
[체크한 항목만 반영] [모두 거부]
```

## 용어 규칙

간단 모드 화면·상태바에는 다음 내부 용어를 노출하지 않는다.

| 내부 | 노출 |
|---|---|
| stage | (숨김, 상태바는 `stageLabel()` 한글명) |
| artifact / trace / packet / schema | "생성 근거 보기" 안의 파일 경로로만 |
| proposal / canon delta | 설정 변경 (후보) |
| candidate | 후보 (원고) |
| resume state | 다음 화 메모 / 재개 상태 |
| fallback | 기본 생성 / 뼈대 |
