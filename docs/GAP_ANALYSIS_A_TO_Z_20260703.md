# A-to-Z 소설 작성 갭 분석 (2026-07-03)

기준 시나리오: 작가가 Bindery만으로 「기획 → 설정 → 집필 → AI 보조 → 검토 → 퇴고 → 연재 출고」를 완주한다고 가정하고, 단계별로 현재 상태와 부족한 부분을 정리한다.

## 이번 개편으로 해소된 것

- **파이프라인 산출물 부재** → 모든 단계(컨텍스트/초안/표현 분석/QA/수정 계획/요약/기록)가 산출물을 남기고, 회차별 보관함에서 열람 가능. 「집필 반영」 산출물은 초안·수정 후보 프롬프트에 자동 포함.
- **AI가 프로젝트 맥락 없이 집필** → `generate_candidate`가 guidance(바이블·문체 지침서·산출물)를 받아 프롬프트에 주입.
- **문체 재현 수단 부재** → `문체` 탭 신설: 샘플 → 장면별 정량 분석 → AI 감성 추출 → 장면별 지침·금지어·금지묘사 → 원문에 없는 재현 샘플 → 최종 문체 지침서(`canon/style-guide.md`).

## 2026-07-03 후속 반영 결과

로컬 저장 범위에서 아래 갭이 해소되었다 (관련 항목에 ✅ 표시):

- ✅ A3 설정집 항목 생성 UI

- ✅ B5 집필 통계 (일별 단어 누적, 내보내기 화면)
- ✅ B6 전체 검색 (⌘⇧F 모달)
- ✅ C7 산출물 파일 동기화 (`.bindery/artifacts/`)
- ✅ C8 요약→바이블 환류 (`canon/summaries/`)
- ✅ C9 QA/수정계획 실체화 (에이전트 기반 + 폴백)

- ✅ D11 스냅샷 복원/비교

- ✅ E13 내보내기 (TXT/HTML/EPUB 합본·회차별 — EPUB은 의존성 없이 STORED ZIP 컨테이너를 직접 조립, `unzip -t` 무결성 검증)

추가: AI 집필 파라미터(분량·창의성·추가 지시), 문체 적용 강도(유연/균형/엄격).

## 남은 갭 (우선순위순)

### A. 기획·구조 (상)
1. **회차 관리 부재** — ep001 하나만 실질 지원. 새 회차 생성/이동/상태(초고·퇴고·발행) 관리 UI가 없다. 연재물 도구로서 최우선 갭.
2. **플롯 보드 읽기 전용** — 장면 추가/드래그/긴장도 편집이 불가. 뮤블 플롯 캔버스, 펜시브 플롯보드 대비 가장 큰 차이.
3. **설정집 항목 생성·편집 UI 부재** — 파일을 직접 만들어야 한다. 자료 화면에서 인물/장소 템플릿 생성 버튼이 필요.

### B. 집필 (중)
4. **회차 내 장면(비트) 구조 부재** — manuscript.md 단일 버퍼. 장면 단위 접기/이동이 없다.
5. **집필 목표·통계 대시보드 부재** — 세션 목표는 있으나 일별 누적, 회차별 분량 추이가 없다.
6. **검색 부재** — 프로젝트 전체 텍스트 검색(⌘⇧F)이 없다.

### C. AI 파이프라인 (중)
7. **산출물이 로컬 스토리지에만 저장** — `.bindery/artifacts/`로 파일 동기화가 없어 다른 기기/에디터에서 볼 수 없다.
8. **요약→시리즈 바이블 자동 반영 부재** — summarize 산출물이 설정집(인물 상태 변화)으로 환류되지 않는다.
9. **QA/수정계획의 네이티브 구현이 스텁** — run_qa는 기존 파일 폴백, generate_revision_plan은 고정 텍스트. novelctl 또는 run_agent_text 기반 실제 생성으로 교체 필요.
10. **문체 일관성 검사 부재** — 지침서로 쓰기는 하지만, 쓴 원고가 지침을 지켰는지 검사하는 게이트(QA 연동)가 없다.

### D. 검토·퇴고 (하)
11. **스냅샷 복원 부재** — 목록·생성만 있고 되돌리기/비교가 없다.
12. **회차 간 연속성 검사 부재** — QA가 단일 회차 기준. 이전 회차 요약 대비 모순 검출이 없다.

### E. 출고 (상)
13. **실제 내보내기 부재** — `내보내기` 탭에 TXT/EPUB/HWP/플랫폼용 포맷 출력이 없다. 이름과 달리 스냅샷·기록 확인 화면에 머물러 있다. (뮤블은 EPUB, 펜시브는 HWP/EPUB 지원)

### F. 기반 (지속)
14. Windows 러너 검증, Vite 500kB 청크 경고, 패키지된 .app 내부 클릭스루 자동화.

## 권장 다음 스프린트

1. 회차 관리(생성·전환·상태) + 내보내기(TXT/EPUB 최소) — 연재 A-to-Z의 시작과 끝을 막는 두 갭.
