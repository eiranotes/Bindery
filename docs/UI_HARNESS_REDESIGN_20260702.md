# UI 전면 개편 — AI 소설 하네스 중심 (2026-07-02)

## 목표

- 작성 화면과 AI 작업을 완전히 분리한다.
- AI 작업을 「연결 → 바이블 → 실행 → 검토」의 단계형 하네스로 재구성해, 처음 보는 사람도 소설 파이프라인 전체 흐름을 한눈에 읽을 수 있게 한다.
- 카드형 CTA를 줄이고 펜시브(Pensiv) 계열의 라인/폼 기반 미니멀 UI로 통일한다.
- 모든 메뉴를 한국어로 유지한다.

## 벤치마크

- **펜시브(pensiv.so)**: 조용한 라인 기반 에디터, 원고 옆 AI 리뷰 패널, 구상→구조화→집필→검토→내보내기 파이프라인, 화면당 하나의 primary 액션.
- **뮤블(muvel.app)**: 회차/플롯/위키/메모의 역할 분리, 사이드바 중심 네비게이션, 자동 저장, 필요한 기능만 배치하는 위젯 철학.

## 정보 구조 (변경 후)

상단 탭 4개: `집필 · 자료 · AI 작업 · 내보내기`
상단 우측: AI 연결 상태 버튼(→ AI 작업 01 연결), `도움말`, `환경설정`, 테마 전환.

| 화면 | 역할 |
| --- | --- |
| 집필 | 에디터만. AI 진입점 없음. 슬래시 AI 명령은 AI 작업 화면으로 안내만 함 |
| 자료 | 설정집(코덱스) + 플롯 보드 |
| AI 작업 | 하네스: 01 연결(CLI 설정·테스트·데모 모드) → 02 바이블(감지/템플릿 생성/생략) → 03 실행(컨텍스트→초안 후보→표현 분석→QA→수정 계획→요약→기록, 단계별 상태·프롬프트 미리보기·연결선) → 04 검토(후보 diff·QA·수정 계획·반복 표현) |
| 내보내기 | 스냅샷 + 실행 기록 |
| 환경설정(모달) | 폰트, 크기, 자동 저장, 줄 번호, 설정 언급, 스마트 입력 |

## 주요 변경

- 신규: `AIStudio.svelte`(하네스 화면), `pipelineStore.ts`(단계 상태), `PreferencesModal.svelte`.
- 재작성: `MainSurface`, `TopBar`, `BinderPanel`(가짜 진행도/회차 목업 제거, 파일·설정집 2탭), `HelpSurface`, `CommandPalette`, `uiStore`.
- 삭제(죽은 코드): `AIPanel`, `SettingsPanel`, `EvidenceSummary`, `RightDock`, `BuildLadder`, `EpisodeTabs`, `buildLadderStore` 및 관련 CSS 블록.
- 검토(구 `검토` 탭)는 AI 작업 04 검토 단계로 통합해 중복 배치를 제거.
- 새로고침 시 열려 있던 프로젝트(파일 트리·원고)를 자동 복원.
- JobConsole의 `\n` 리터럴 표시 버그 수정.
- 바이블이 없는 프로젝트: 템플릿 생성(`canon/setting-bible.md`) 또는 「바이블 없이 진행」 명시 선택 지원.
