# 펜시브(pensiv.so) UI 벤치마크 (2026-07-05)

> **적용 노트 (2026-07-05)**: §2 대조표의 "Bindery 현재"는 레거시 기준이다. 신규 구축에서는
> §3의 결론이 그대로 구현되었다 — 전 구간 제안→승인 관문(제안·정사 화면), 좌측 탐색의 소재/제안
> 진입점, 인스펙터의 대기 결정 노출. 시트/캔버스/작업 관리는 roadmap Phase 2에 있다.


Bindery의 UI 기준 제품인 한국 웹소설 집필 플랫폼 **펜시브(pensiv)** 를 웹에서 재조사한 결과와,
Bindery에 이미 흡수된 부분/아직 흡수하지 못한 부분의 대조표다.

조사 경로: 공식 사이트(pensiv.so/ko), 나무위키 항목, 작가 커뮤니티(아카라이브 웹소설 채널) 공개 글.
주의: pensive.me(미국 AI 집필 스튜디오)는 별개 제품이며 벤치마크 대상이 아니다(과거 혼동 이력 있음 — state.md 2026-07-03).

## 1. 펜시브 구조 요약

### 화면 구조
- **좌측**: 프로젝트 구조 사이드바 — 문서 트리 + 파일 타입 아이콘 + 작업(Task) 표시
- **상단**: 열린 문서 탭 — 여러 문서를 탭으로 병행 작업
- **중앙**: 현재 파일/보드 작업 영역 (고성능 에디터 중심)
- **우측/보조**: AI 리뷰 패널 — 피드백을 문맥 안에서 제공. "계획 → 확인 → 생성" 순서가 기본 흐름
- **하단/위젯**: 타이머, 단어 목표, 작업 관리

### 파일 타입 (핵심 벤치마크 대상)
| 타입 | 역할 |
|---|---|
| 문서 | 본문 집필 |
| 시트 | 표 형식 데이터(인물표, 설정표) |
| 플롯보드 | 비트/전개를 카드로 구성 |
| 캔버스 | 장면·인물·세계관의 시각 배치(그래프적 사고) |
| 노트 | 참고 자료, 아이디어 |

### 기능
- 백링크 기반 연결("링크는 곧 맥락") + 그래프 뷰
- 1년 버전 기록 추적
- 우선순위/날짜 기반 작업 관리, 문서·프로젝트 전체 검색
- 글쓰기 통계(단어 목표, 타이머)
- 내보내기: Word/PDF/EPUB/HWP/Markdown
- AI: 결과물을 바로 생성하지 않고 작가의 계획을 먼저 확인하는 흐름 — Bindery의 candidate-first 원칙과 동일한 방향
- 기본 기능 무료, AI 기능만 유료화 예정

### 한국어 UI 톤
- 밀도 높은 작업 도구형 문구, 짧은 명사구 라벨
- "생각이 앞서도 끊기지 않게" 같은 기능 본질 중심 카피
- 카드 장식보다 표·레일·인스펙터 위주 (Bindery DESIGN.md anti-slop 규칙과 일치)

## 2. Bindery 대조표

| 펜시브 요소 | Bindery 현재 | 처분 |
|---|---|---|
| 좌측 구조 사이드바 | BinderPanel(회차/파일/바이블 탭) | 유지. 소재(ideas)·제안(proposals) 진입점 추가 |
| 상단 탭 작업 흐름 | 상단 3탭(초벌·다듬기·자료)+작업실 | 유지. 구상(소재·세계관) 진입 추가 |
| 문서 | MarkdownEditor(CodeMirror) | 유지 |
| 시트 | 없음(코덱스 항목이 근사) | roadmap: 인물/설정 시트 뷰 |
| 플롯보드 | PlotGridPanel + episode-outline | 유지 |
| 캔버스/그래프 뷰 | 없음 | roadmap (docs/TASKS.md 기존 항목과 병합) |
| AI 리뷰(우측 패널) | 파이프라인 워크벤치 인스펙터 + QA/검토 탭 | 유지. proposal inspector 추가 |
| 계획→확인→생성 | outline/브리프/장면계획 승인 게이트 | 이번 작업에서 proposal 승인으로 확장 |
| 버전 기록 | snapshot + run 기록 | 유지 |
| 작업 관리 | 없음(외부 docs/TASKS.md) | roadmap |
| 글쓰기 통계 | statsStore(공백 제외 자수 등) | 유지 |
| 내보내기 | TXT/HTML/EPUB | 유지. HWP는 범위 외 |

## 3. 이번 작업에 반영하는 구조 결론

1. 펜시브의 "계획 → 확인 → 생성"을 제품 전 구간으로 확장: 소재→세계관→정사 갱신까지 모두 제안(proposal)→사람 승인→반영.
2. 좌측 사이드바에 파이프라인 이전 단계(소재/구상)가 보여야 신규 작품 흐름이 끊기지 않는다.
3. 우측 인스펙터에 "대기 중 proposal" 카운트를 노출해 승인 대기 작업이 침묵 속에 쌓이지 않게 한다.
4. 캔버스/시트/작업 관리는 이번 범위에서 제외하고 roadmap에 남긴다(docs/implementation/roadmap.md).

## 4. Open Design MCP 사용 기록

Open Design MCP 데몬은 연결 가능했으나 프로젝트가 비어 있었다(펜시브 참조 자산 없음).
Open Design은 로컬 디자인 워크스페이스라 외부 사이트 화면을 직접 수집할 수 없어, 화면 구조 분석은 위의 웹 조사로 수행했다.
대신 목표 레이아웃 와이어프레임을 Open Design 프로젝트(`bindery-harness-ui`)로 생성해 토큰/레이아웃 방향을 기록했다(§docs/design/ui_direction.md 참조).
과거 세션에서는 Open Design editorial/warm-editorial 레퍼런스로 DESIGN.md 토큰을 정리한 이력이 있다(docs/TASKS.md 2026-07-04).
