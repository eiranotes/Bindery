# AI 증강 집필 워크플로우 (지시서 §4 대응)

전제: 설정 화면에서 CLI 실행기 연결(프리셋 codex/claude/gemini/직접 설정) 후 연결 테스트 통과.
모든 AI 출력은 후보/제안/보고서로만 남고, 원본 확정은 사람의 액션이다.

## 1. 소재 발굴 (§4.1)
소재 화면에서 장르/분위기/클리셰/독자 경험/금지 요소/후보 수 입력 → **소재 후보 생성**.
- 후보는 `ideas/inbox/*.md` 파일 카드로 쌓인다 (채팅 응답 아님).
- 「선별 권고(AI)」는 서열·근거 보고서만 만든다. 채택/보류/폐기는 카드 버튼(=폴더 이동).
- CLI가 없으면 「웹 AI packet」→ ChatGPT/Claude/Gemini 웹에 붙여넣기 → 응답 JSON을
  「웹 AI 결과 가져오기」에 붙여넣으면 스키마 검증 후 inbox에 등록된다.

## 2. 세계관 확장 (§4.2)
세계관 화면에서 채택 소재 기반 **확장 proposal 생성** → 제안·정사 화면으로 이동.
- 자산(인물/장소/기관/체계/규칙/용어/관계/갈등)별로 승인/거부 → **결정 반영** 시에만 파일 생성.
- 거부 항목은 파일이 되지 않고 이력으로 남는다.

## 3. 바이블 생성 (§4.3)
세계관 화면 **자산에서 바이블 조립** → 후보 미리보기 → **이 후보로 교체** (기존 바이블 스냅샷).
바이블은 요약층, 자산 파일이 상세층으로 유지된다.

## 4. 플롯 제작 (§4.4)
플롯 화면 **회별 계획 제안** (아크 + 회차별 목적/beats/떡밥 심기·회수/hook/risk).
- 표에서 직접 수정 가능, 회차별/전체 승인. 승인 회차는 재제안이 덮지 못한다.
- 산출: `plot/plot-board.json` + `plot/series-outline.md`.

## 5. 회차 설계 (§4.5)
회차 화면 「계획」 탭: **브리프 생성** (승인 플롯 row + resume state + 바이블 + 이전 회차 확정
원고 끝부분이 입력) → **장면 계획 생성** (브리프가 hard constraint).
둘 다 회차 폴더의 파일로 남고 직접 수정 가능. 브리프·장면 계획 없이는 초안 버튼이 잠긴다.

## 6. 원고 후보 생성 (§4.6)
「계획」 탭에서 후보 수(1~4)와 지시를 넣고 **초안 후보 생성**.
- 후보는 `.bindery/candidates/{ep}/`에만 쓰인다. 원고는 그대로다.
- 「후보·적용」 탭에서 후보 선택 → 현재 원고 대비 diff → hunk 선택 → **선택 적용** (스냅샷 선행).
- 후보가 분리해낸 설정 변경 후보(canon_delta_candidates)는 카드에 표시되고 §8로 이어진다.

## 7. QA와 수정 (§4.7)
「QA·수정」 탭에서 관점별 실행: 문체·리듬·훅 / 플롯·연속성(장면 기능 반복, 떡밥 진행 포함) /
정사·설정 충돌. 게이트별 점수·판정·본문 인용 근거가 남는다.
- **수정 계획 생성** → 항목별 수용/기각 체크(사람) → **수정 후보 생성** → diff에서 일부만 적용 가능.
- 재-QA로 루프를 닫는다.

## 8. 요약과 정사 갱신 (§4.8)
「요약·픽스」 탭: **요약 생성**(장면 흐름/인물 상태/관계/떡밥/다음 연결점) → `canon/summaries/{ep}.md`.
- **정사 변경 제안** → CanonDeltaProposal (파일별 patch + 위험도) → 제안·정사 화면에서
  항목별 승인 → 반영(스냅샷 + 표식 있는 append).
- **픽스** → 원고 스냅샷 + `.bindery/episodes.json` 갱신 + `status/resume-state.md` 재생성
  (위치/최근 요약/인물 상태/열린 떡밥/보류 중 결정).
- 홈 화면이 다음 회차 진입점을 안내한다. 다음 브리프는 resume state를 입력으로 받는다.

## 투명성
- 실행마다: 하단 상태바에 현재 실행 stage/stdout tail/취소, 하단 도크에 세션 기록,
  `.bindery/runs/`·`.bindery/trace/`에 영구 기록.
- 프롬프트 원본은 `prompts/*.prompt.md` — 파일을 열면 각 단계가 무엇을 지시받는지 그대로 보인다.
