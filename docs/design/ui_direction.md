# Bindery UI 방향 (2026-07-05, autopilot 개편 반영)

레거시 UI를 답습하지 않고 새로 구축했다. 구조 기준은 펜시브 벤치마크
(docs/research/pensiv_ui_benchmark.md)에서 출발했으나, 같은 날 UX-first autopilot 개편으로
"파이프라인 노출형"에서 "자동화+최종 선택형"으로 재정렬했다
(bindery_ux_autopilot_pipeline_plan.md). 시각 토큰은 DESIGN.md의 warm-paper/teal 계약을
따른다(src/app.css가 단일 정의처).

## 1. 전체 프레임 — 최대 2단

```text
상단   : 브랜드 · 작품명 · 모드 탭 · 실행기 상태 (설계자 모드에서만 run 토글)
중앙   : 현재 모드의 작업면 — 단일 컬럼, 중앙 정렬(max 860px), 크게
하단   : (설계자) run 로그 도크 토글
최하단 : 1줄 상태바 — 실행 중 단계(한글명)/stdout tail/취소, 보류 수, 실행기 (설정에서 숨김)
```

- **좌측 사이드바 없음, 우측 고정 인스펙터 없음.** 3단 배치 금지.
- 2단 분할은 작품노트(목록 240px + 뷰어) 한 곳만 허용.
- 와이어프레임: Open Design `bindery-harness-ui/autopilot-shell.html` (S1~S5).

## 2. 모드 이원화

### 간단 모드 (기본)

| 메뉴 | 구조 | 핵심 규칙 |
|---|---|---|
| 홈 | 다음 작업 센터 — CTA 1개 + 한 줄 메모 + 보조 카드 | workflow가 계산한 행동 하나만 크게 |
| 집필 | 진행 레일(설계·초안·검토수정·마감) + 상황별 블록 1개 | 사용자는 단계를 조작하지 않는다. 실행 중엔 스트리밍 패널 |
| 작품노트 | 세그먼트(바이블·인물·세계·플롯·떡밥·스타일) + 목록 + 에디터 | 파일이 진실, 즉시 편집·저장 |
| 보류함 | proposal 체크리스트 (AI 추천 기본 체크) | 체크 반영/모두 거부, 반영 전 스냅샷 고지 |
| 파일 | 트리 + CodeMirror + 스냅샷 복원 | 안전판 |
| 설정 | 화면 모드 + CLI 프로필 + 연결 테스트 | provider 비고정 |

간단 모드 금지 노출: stage id, packet, trace, artifact, schema, canon delta, proposal raw,
브리프/장면 계획 승인 버튼, 개별 QA 버튼. 전부 "생성 근거 보기" 접힘 또는 설계자 모드로.

### 설계자 모드 (설정에서 전환)

간단 메뉴 + 소재/세계관/플롯/회차/제안·정사 + run 도크. 기존 파이프라인 화면 그대로.

## 3. 집필 화면 상태 기계

```text
실행 중        → LiveRunPanel (CLI stdout 실시간, 단계 경계 표시, 취소)
원고 X 후보 X  → 한 줄 메모 + [이번 화 쓰기]
원고 X 후보 O  → 후보 카드 2~3 (라벨·요약·위험·추천) [미리보기|이 후보 적용]
미리보기       → 전문(serif) ↔ diff 토글 + 적용
원고 O         → 원고 리더(serif, 주인공) + [검토하고 수정안] [직접 고치기] [회차 마감]
수정 제안      → 체크리스트(추천 기본 체크) + [체크한 수정 적용]
마감 카드      → 설정 변경 후보 체크(위험 높음 해제) + [반영하고 마감] [보류]
```

## 4. 정직성 규칙 (전 화면 공통, 유지)

- AI/뼈대/웹 교환 출처 칩 표시. 오프라인 뼈대는 AI 흉내 금지.
- 파괴 위험 액션(후보·수정 적용, 정사 반영, 마감)은 스냅샷 선행을 문구로 고지.
- 산출물 카드·근거 보기에는 실제 파일 경로 표시.
- 텍스트가 칸보다 길면 세로로 밀리지 않게: 요약 우선, 전문은 미리보기/접힘, 긴 출력은
  자체 스크롤 컨테이너.

## 4.5 Context Pack 투명성 (2026-07-06, 컨셉 정의서 §6.2·§7.3)

- 산출물(초안 후보) 옆에 "참고: … / 비어 있어 추정: …" 한 줄 요약을 붙인다.
  전체 Context Pack 원문을 펼치는 것이 아니라 핵심 근거와 대체 항목만 짧게.
- "생성 근거 보기"의 최상단은 자료별 목록이다: 투입/비어 있음 칩 + 자료명 +
  투입 방식(핵심 6,000자·이번 화 row만·요약 우선·끝부분 2,000자만) + mono 경로.
  데이터는 `loadEpisodeContext().sources`가 단일 출처.
- 검토 결과는 QA 3관점(설정·연속성·문체) overall verdict 칩(통과/주의/수정 필요)으로
  먼저 요약하고, 근거 원문은 근거 보기 접힘에 둔다.
- 프리플라이트는 질문이 아니라 선택지다: hard gate(바이블·이번 화 플롯)는 준비 CTA로 막고,
  soft gap(스타일 지침·이전 요약)은 시작 블록에 알린 뒤 "작품노트에서 채우기" 또는
  "그대로 진행(AI 추정)"으로 계속 굴러가게 한다.
- 되돌리기는 명시적 버튼이다: 원고 화면 [이전본 복원]이 최신 원고 스냅샷으로 복원하고,
  복원 전 상태도 스냅샷으로 남긴다.
- 후보 비교(초안·기획)는 좁은 세로 카드 대신 전폭 행 리스트 — 긴 요약이 세로로 늘어지지
  않고 줄 단위로 비교된다. 원고 리더/에디터는 min(66vh, 720px)로 화면의 중심.

## 5. 시각 언어 (유지)

- 카드 중첩 금지: hairline 구분의 표/레일/디테일 행.
- 라이트(warm paper) 기본, 다크 토큰 제공.
- 본문(원고 리더·미리보기)만 serif, 도구 UI는 Pretendard 13.5px 밀도.
- 강조는 teal 하나, 상태는 ok/warn/bad 3색.
- 이모지 아이콘·히어로 카드·그라데이션 금지.
