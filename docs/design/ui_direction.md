# Bindery UI 방향 (신규 구축, 2026-07-05)

레거시 Bindery(HermesWorkspace/project/Bindery)의 UI를 답습하지 않고 새로 구축했다.
구조 기준은 펜시브 벤치마크(docs/research/pensiv_ui_benchmark.md), 시각 토큰은 레거시 DESIGN.md의
warm-paper/teal 계약을 흡수했다(src/app.css가 단일 정의처).

## 1. 전체 프레임

```text
상단   : 브랜드 · 작품명 · 모드 탭(홈·소재·세계관·플롯·회차·제안·정사·파일·설정)
        · busy 표시 · 실행기 상태 · run 기록 토글
좌측   : 구조 탐색 — 소재 상태별 수, 자산 수, 플롯 승인 수, 회차별 진행 상태, 제안 대기 수
중앙   : 현재 모드의 작업면
우측   : 인스펙터 — 최근 실행 투명성(AI/폴백, 시간, 프롬프트 파일, stderr), 대기 중 결정, 원칙 고지
하단   : run 로그 도크 (토글) — 세션 실행 기록, 전체는 .bindery/runs/
```

모드가 그대로 제작 흐름이다: 소재 → 세계관 → 플롯 → 회차 → 제안·정사 → (다음 회차).
홈은 재개(resume) 지점이고, 파일은 Markdown-native 원칙의 안전판(모든 산출물 직접 수정)이다.

## 2. 모드별 작업면

| 모드 | 구조 | 핵심 규칙 |
|---|---|---|
| 홈 | 다음 작업 목록 + resume-state 본문 | 상태에서 다음 행동을 계산해 제시 |
| 소재 | 조건 폼 / 상태 필터(inbox·씨앗·채택·폐기) / 카드 목록+상세 / 웹 AI 교환 | 버튼 = 폴더 이동. AI는 inbox에만 쓴다 |
| 세계관 | 확장 proposal 생성(+packet) / 자산 파일 목록·뷰어 / 바이블 후보→적용 | 승인 없이 파일 생성 금지, 바이블 교체 전 스냅샷 |
| 플롯 | 회별 계획 표(직접 편집) + 회차별 승인 | 승인 회차는 재제안이 못 덮음 |
| 회차 | 탭: 계획 / 후보·적용(diff hunk) / QA·수정 / 요약·픽스 | 브리프+장면 없이 초안 금지, 적용 전 스냅샷 |
| 제안·정사 | proposal 목록 → 항목별 승인/거부 → 결정 반영 | AI→canon의 유일한 관문 |
| 파일 | 트리 + 플레인 에디터 | 무엇이든 열어 고칠 수 있다 |
| 설정 | CLI 프로필(명령/인자 템플릿/모델/타임아웃/오프라인) + 연결 테스트 | provider 비고정, {prompt}/{model} 치환 |

## 3. 정직성 규칙 (전 화면 공통)

- AI 산출과 로컬 폴백을 항상 구분 표시: `AI` / `로컬 폴백` / `웹 교환` 칩 + 폴백 사유.
- 오프라인 뼈대는 "AI 흉내"를 내지 않는다 — 뼈대임을 본문과 리스크에 명시.
- 파괴 위험 액션(후보 적용, 바이블 교체, proposal 반영)은 반드시 스냅샷 선행을 UI 문구로 고지.
- 모든 산출물 카드에는 실제 파일 경로를 표시한다.

## 4. 시각 언어

- 카드 중첩 금지: hairline 구분의 표/레일/디테일 행.
- 라이트(warm paper) 기본, prefers-color-scheme 다크 토큰 제공.
- 본문(원고·읽기)만 serif, 도구 UI는 Pretendard 계열 13.5px 밀도.
- 강조는 teal(--accent) 하나, 상태는 ok/warn/bad 3색만.
- 이모지 아이콘·히어로 카드·그라데이션 금지 (레거시 anti-slop 계약 유지).

## 5. Open Design 기록

작업 초기 Open Design MCP로 와이어프레임(`bindery-harness-ui` 프로젝트, harness-wireframes.html)을
생성해 구상 화면·제안 검토 레이아웃을 잡았다. 이후 구현은 이 문서의 구조를 기준으로 진행했다.
