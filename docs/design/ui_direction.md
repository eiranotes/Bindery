# Bindery UI 방향 — AI 장기 집필 하네스 (2026-07-05)

DESIGN.md(토큰 계약)와 docs/research/pensiv_ui_benchmark.md 위에서, 하네스 심화 작업이 추가/변경하는 UI 구조를 정의한다.
색·타이포·간격 토큰은 DESIGN.md가 유일한 기준이며 여기서는 구조만 다룬다.

## 1. 전체 프레임 (유지 + 확장)

```text
상단   : 프로젝트 제목 · 상위 탭(구상 · 초벌 · 다듬기 · 자료 · 작업실) · 실행기 상태
좌측   : BinderPanel — 회차 / 파일 / 바이블 (+ 소재·제안 진입은 구상/초벌 화면 내부 레일)
중앙   : 현재 화면의 작업면 (보드/워크벤치/에디터)
우측   : 화면별 인스펙터 (실행 정보 · 입력 기준 · proposal 대기 수)
하단   : (기존) run 기록·산출물은 파이프라인 워크벤치 내부 탭으로 유지
```

변경점은 상위 탭에 **구상(ideate)** 이 추가되는 것 하나다. 나머지는 기존 화면 내부 확장.

## 2. 구상 화면 (신규)

소재 발굴 → 세계관 확장이 한 화면에서 이어진다.

```text
+--------------------------------------------------------------+
| 구상 | 조건 입력(장르/분위기/금지요소) [소재 후보 생성] [packet] |
+------------------+-------------------------------------------+
| 소재 목록(레일)   |  선택 소재 본문 (Markdown 미리보기)          |
|  inbox (n)       |  hook / 감정 엔진 / 독자 약속 / 리스크        |
|  seeds (n)       |                                           |
|  selected (n)    |  [씨앗으로] [채택] [보류] [폐기] [파일 열기]   |
|  discarded (n)   +-------------------------------------------+
|                  |  세계관 확장 (채택 소재 기반)                 |
|                  |  [확장 proposal 생성] → 제안 자산 목록        |
|                  |  자산별 [승인→파일 생성] [보류] [거부]         |
+------------------+-------------------------------------------+
```

- 소재 카드는 `ideas/{inbox,seeds,selected,discarded}/*.md` 파일과 1:1. 목록은 파일 시스템이 진실.
- 모든 버튼은 파일 이동/생성일 뿐이며, 사용자는 언제든 파일을 직접 열어 수정할 수 있다.

## 3. 파이프라인 워크벤치 확장

- 중앙 탭에 **제안(proposals)** 추가: 산출물 · 프롬프트 · 컨텍스트 · 작품 플롯 · 검토 · **제안**
- 제안 탭: 대기 중 proposal 목록(세계관 확장/정사 변경) → 변경 항목별 대상 파일 · patch 미리보기 · 위험도 · [승인] [거부]
- 우측 인스펙터 "위험 · 결정" 섹션에 `제안 대기 n건` 행 추가 (0이 아니면 accent 표시)
- summarize 단계가 정사 변경 proposal을 만들면 토스트로 제안 탭을 가리킨다.

## 4. 재개(resume) 노출

- 파이프라인 타깃 밴드 아래에 `status/resume-state.md` 존재 시 "재개 상태 갱신: {시각}" 표시.
- resume state는 commit(기록·픽스) 단계가 재생성하며, 회차 브리프/장면 계획 프롬프트의 원천 문서 목록에 포함된다.

## 5. 프롬프트 투명성

- 신규 단계(소재 발굴/세계관 확장/정사 delta)는 blueprint 파일(`src/lib/prompts/*.prompt.md`)에서 조립되며,
  프롬프트 미리보기에서 "blueprint: 파일명" 출처를 표시한다.
- 기존 9단계 프롬프트의 blueprint 이관은 roadmap 항목.

## 6. 톤과 밀도

- 펜시브식 밀도: 카드 중첩 금지, 표/레일/hairline 구분 유지 (DESIGN.md anti-slop 준수)
- 상태 라벨은 2~3글자 한국어(채택/보류/폐기/승인/거부), 위험도는 low/medium/high를 낮음/중간/높음으로.
- AI 폴백 산출물은 항상 `(로컬)` 접미사와 warn 톤으로 정직하게 표시(기존 규칙 유지).

## 7. Open Design 와이어프레임

Open Design MCP 프로젝트 `bindery-harness-ui`에 구상 화면과 제안 탭 와이어프레임을 아티팩트로 남겼다.
토큰은 DESIGN.md의 warm-paper/teal 계약을 그대로 쓴다(신규 토큰 없음).
